# Ручная проверка self-hosted установки

Этот документ нужен для спокойной ручной проверки после установки `Интеллион Метрика`.

Здесь нет внутренней разработки и служебной кухни. Только то, что нужно проверить руками.

## Что должно быть уже готово

- установка завершилась без критической ошибки;
- у вас есть адрес панели;
- у вас есть ссылка активации владельца;
- домен уже смотрит на сервер.

## Шаг 1. Проверить, что панель открывается

Откройте адрес панели в браузере.

Если выбран path-режим:

- `https://example.com/analytics`

Если выбран режим отдельного поддомена:

- `https://analytics.example.com`

Что должно получиться:

- страница входа открывается;
- нет ошибки сертификата;
- нет ошибки `502` или `404`.

## Шаг 2. Активировать владельца

На сервере откройте:

```bash
cat /opt/intellion-metrica/state/owner-activation.txt
```

Перейдите по ссылке и задайте пароль владельца.

Что должно получиться:

- пароль задается без ошибки;
- после этого владелец может войти в панель.

## Шаг 3. Войти в Метрику

Войдите под почтой владельца и новым паролем.

Что должно получиться:

- открывается панель Метрики;
- доступны разделы `Обзор` и `Управление`;
- владелец видит настройку сайтов.

## Шаг 4. Добавить сайт

В панели:

1. откройте `Управление`;
2. перейдите в `Сайты`;
3. добавьте свой сайт.

Минимально заполните:

- код сайта;
- название;
- основной домен;
- разрешенные домены.

Что должно получиться:

- сайт сохраняется;
- он появляется в списке;
- можно открыть его настройки повторно.

## Шаг 5. Включить безопасный сбор данных

После создания сайта включите безопасный сбор данных и сохраните настройки.

Проверьте:

- у сайта есть ключ;
- доступен секрет для подписанного приема данных;
- есть адреса для отправки событий и согласий.

## Шаг 6. Подключить сайт

Подключите свой сайт к Метрике через маршрут, который указывает панель.

После подключения откройте сайт в браузере и выполните несколько обычных действий:

- открытие страницы;
- переход между страницами;
- нажатие на основную кнопку;
- отправка формы, если она есть.

## Шаг 7. Проверить, что события пошли

Вернитесь в панель и откройте `Обзор`.

Что должно получиться:

- появляются посещения;
- фиксируются события;
- видна активность по сайту.

Если включен учет согласий, проверьте также их фиксацию.

## Шаг 8. Проверить цели

Добавьте хотя бы одну простую цель и вызовите соответствующее действие на сайте.

Что должно получиться:

- цель сохраняется;
- после тестового действия цель начинает фиксироваться.

## Шаг 9. Быстрый чек-лист

Считать ручную проверку успешной можно, если:

- панель открывается;
- владелец активирован;
- вход работает;
- сайт добавлен;
- события поступают;
- согласия фиксируются, если включены;
- цели можно создать и проверить;
- нет явных ошибок в браузере и в панели.

## Частые проблемы

### Панель не открывается

Проверьте:

- домен направлен на сервер;
- открыты порты `80` и `443`;
- установка завершилась успешно;
- контейнеры запущены.

### Не проходит активация владельца

Проверьте:

- вы открыли актуальную ссылку из `owner-activation.txt`;
- установка не была прервана;
- время на сервере выставлено правильно.

### События не появляются

Проверьте:

- сайт действительно подключен к Метрике;
- код сайта и домен указаны верно;
- безопасный сбор данных включен;
- тестовые действия на сайте действительно были выполнены после подключения.

## Что уже подтверждено, а что еще нужно проверить

Уже подтверждено:

- install channel через GitHub;
- запуск installer по прямой ссылке;
- скачивание install bundle из GitHub Releases;
- наличие образов в GHCR;
- работа owner activation flow в продукте.

Еще нужен отдельный живой end-to-end прогон на чистом внешнем сервере:

- полный self-hosted install по этой инструкции;
- первое подключение реального сайта;
- ручная проверка сбора событий после публикации в боевом домене.
