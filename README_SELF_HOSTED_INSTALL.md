# Как установить Интеллион Метрика у себя

## Что это за продукт

`Интеллион Метрика` — self-hosted система аналитики сайта.

После установки вы получаете:

- отдельную панель Метрики;
- владельца установки;
- возможность добавить свои сайты;
- безопасный сбор событий и согласий;
- отчеты, цели и базовый рабочий контур.

Метрика работает на вашем Linux-сервере и публикуется либо по пути вида `/analytics`, либо на отдельном поддомене.

Дополнительный документ для ручной приемки после установки:

- `README_SELF_HOSTED_MANUAL_TESTS.md`

## Что нужно до начала установки

Минимально нужно:

- Linux-сервер с SSH-доступом;
- доступ `root` или `sudo`;
- публичный домен или поддомен для панели;
- почта владельца Метрики;
- сайт, который вы будете подключать к Метрике.

По серверу:

- память: минимум `2 GB`, желательно `4 GB` и выше;
- диск: минимум `8 GB`, желательно `15 GB` и выше;
- открытые `80` и `443` для публикации панели;
- исходящий доступ в интернет для загрузки образов.

Docker устанавливать вручную не обязательно: установщик проверит его сам и при необходимости поставит автоматически.

## Какой вариант публикации выбрать

### Основной вариант: на основном домене по адресу `/analytics`

Пример:

- `example.com/analytics`
- `site.ru/analytics`

Это можно выводить на первый план как основной сценарий для покупателя.

Плюсы:

- не нужен отдельный поддомен;
- проще для клиента в первом запуске;
- адрес панели остается на основном домене сайта.

### Отдельный поддомен

Пример:

- `analytics.example.com`
- `analytics.site.ru`

Этот вариант можно оставлять:

- если у вас уже есть поддомен;
- если вы хотите полностью развести сайт и панель;
- как отдельный или расширенный вариант внедрения.

## Подготовка сервера

### Шаг 1. Подключитесь к серверу

Пример:

```bash
ssh root@<IP_СЕРВЕРА>
```

Если вы работаете не под `root`, нужен пользователь с `sudo`.

### Шаг 2. Проверьте домен

Это значит следующее:

- если вы ставите Метрику по адресу `example.com/analytics`, домен `example.com` должен вести на тот сервер, где вы запускаете Метрику;
- если вы ставите Метрику на поддомене `analytics.example.com`, то на ваш сервер должен вести именно `analytics.example.com`.

Иначе установка может завершиться успешно, но панель не откроется в браузере, потому что запрос уйдет на другой сервер.

Пример:

- `example.com` -> IP вашего сервера
- или `analytics.example.com` -> IP вашего сервера

Проще говоря:

- вы запускаете Метрику на сервере с IP `203.0.113.10`;
- значит и домен панели должен указывать на `203.0.113.10`.

Если DNS еще не настроен, установка может пройти, но панель не будет открываться снаружи.

Если хотите быстро проверить это вручную, можно выполнить на своем компьютере:

```bash
nslookup example.com
```

или:

```bash
ping example.com
```

В ответе должен быть IP именно того сервера, куда вы ставите Метрику.

### Шаг 3. Заранее определите 3 значения

Перед запуском установщика вам нужно просто заранее решить, какие значения вы будете указывать в команде:

- по какому адресу будет открываться панель;
- как будет называться эта установка внутри Метрики;
- какая почта будет у первого владельца панели.

Это не отдельная техническая подготовка.  
Нужно просто заранее выбрать эти значения, чтобы потом подставить их в команду установки.

Пример:

- адрес панели: `example.com`
- имя установки: `Example Analytics`
- почта владельца: `owner@example.com`

Что это значит на практике:

- `адрес панели` — это адрес, по которому вы потом будете открывать Метрику;
- `имя установки` — это внутреннее понятное название этой установки, например название компании или проекта;
- `почта владельца` — это email человека, который станет первым владельцем панели и будет выполнять первый вход.

Важно про владельца:

- сейчас система не отправляет автоматическое письмо на эту почту сразу в момент установки;
- после установки на сервере появляется одноразовая ссылка активации владельца;
- по этой ссылке владелец сам задает пароль и активирует доступ.

То есть в обычном self-hosted сценарии логика такая:

1. вы указываете почту владельца в команде установки;
2. установка завершается;
3. на сервере появляется файл со ссылкой активации;
4. вы открываете эту ссылку сами или передаете ее владельцу;
5. владелец задает пароль и заходит в панель.

Если установку делает Intellions с нашей стороны, мы передаем клиенту уже готовую ссылку активации или включаем ее в handoff после установки.

## Основная команда установки

### Вариант 1. Из install bundle, который вы получили вместе с продуктом

Сначала скачайте install bundle на сервер.

Если Intellions передала вам прямую ссылку на архив, пример такой:

```bash
curl -fLO https://github.com/Intellions-ru/metrica-install/releases/download/v0.1.0/intellion-metrica-install-bundle-v0.1.0.tar.gz
```

После этого распакуйте архив и перейдите в его каталог.

Пример:

```bash
tar -xzf intellion-metrica-install-bundle-<version>.tar.gz
cd intellion-metrica-install-bundle-<version>
```

Основная команда:

```bash
sudo bash ./scripts/install_metrica.sh \
  --publish-mode path \
  --domain example.com \
  --entry-path /analytics \
  --installation-name "Example Analytics" \
  --owner-email owner@example.com
```

### Вариант 2. Запуск по прямой ссылке на установщик

Если вы хотите не распаковывать bundle вручную, можно сразу запустить установщик по публичной ссылке:

```bash
curl -fsSL https://raw.githubusercontent.com/Intellions-ru/metrica-install/v0.1.0/install_metrica.sh | sudo bash -s -- \
  --publish-mode path \
  --domain example.com \
  --entry-path /analytics \
  --installation-name "Example Analytics" \
  --owner-email owner@example.com
```

### Нужен ли Git

Нет, в обычном клиентском сценарии Git не должен быть обязательным.

Нормальный путь для покупателя такой:

- скачать install bundle по прямой ссылке;
- либо запустить installer по прямой HTTPS-ссылке;
- выполнить одну команду установки.

Git или GitHub могут использоваться нами как канал публикации артефактов, но для клиента это не должно быть обязательным шагом.

### Что означают параметры

- `--publish-mode` — режим публикации: `subdomain` или `path`
- `--domain` — адрес панели
- `--installation-name` — человекочитаемое имя этой установки
- `--owner-email` — почта владельца

### Пример для основного варианта

```bash
sudo bash ./scripts/install_metrica.sh \
  --publish-mode path \
  --domain example.com \
  --entry-path /analytics \
  --installation-name "Example Analytics" \
  --owner-email owner@example.com
```

### Пример для отдельного поддомена

```bash
sudo bash ./scripts/install_metrica.sh \
  --publish-mode subdomain \
  --domain analytics.example.com \
  --installation-name "Example Analytics" \
  --owner-email owner@example.com
```

## Что происходит во время установки

Установщик делает это автоматически:

1. Проверяет сервер.
2. Проверяет права доступа.
3. Проверяет память, диск и занятые порты.
4. Проверяет Docker и ставит его, если нужно.
5. Подготавливает рабочие каталоги.
6. Запускает контейнеры базы, API, worker, product control-plane и proxy.
7. Применяет изменения структуры базы.
8. Создает владельца.
9. Подготавливает ссылку активации владельца.
10. Делает базовую post-install проверку.
11. Сохраняет итоговый отчет установки.

От вас в этот момент обычно не требуется ручная сборка или отдельный запуск служб по частям.

## Что вы получаете после установки

После успешного завершения вы получите:

- адрес входа;
- рабочий адрес панели;
- почту владельца;
- ссылку активации владельца;
- статус установки;
- путь к журналу;
- путь к итоговому отчету.

Основные файлы на сервере:

- `/opt/intellion-metrica/state/owner-activation.txt`
- `/opt/intellion-metrica/logs/install-<timestamp>.log`
- `/opt/intellion-metrica/artifacts/install/<timestamp>-one-command-install/install-summary.json`

Что должно получиться:

- точка входа `/analytics` открывается по вашему адресу;
- файл `owner-activation.txt` существует;
- в `install-summary.json` статус установки `passed` или `passed_with_warnings`.

## Первый вход владельца

### Шаг 1. Откройте ссылку активации

После установки откройте файл:

```bash
cat /opt/intellion-metrica/state/owner-activation.txt
```

Внутри будет одноразовая ссылка активации.

### Шаг 2. Задайте пароль владельца

Откройте ссылку в браузере и задайте пароль.

Что должно получиться:

- владелец активирован;
- дальше вы входите в Метрику обычным логином и паролем.

### Если ссылка активации истекла

На текущем этапе это не self-service сценарий внутри панели.

Если ссылка истекла или потеряна:

- обратитесь в Intellions;
- либо к тому, кто сопровождал установку с нашей стороны.

### Если доступ утерян

Базовый путь такой же:

- не пытайтесь пересоздавать установку;
- обратитесь в Intellions для перевыпуска или восстановления owner access.

## Как подключить свой сайт к Метрике

Это основной следующий шаг после первого входа.

### Шаг 1. Войдите в панель

Откройте адрес входа и войдите как владелец.

### Шаг 2. Добавьте сайт

В панели:

- откройте раздел `Управление`
- перейдите в `Сайты`
- нажмите `+`
- создайте сайт

Заполните минимум:

- код сайта;
- название;
- основной домен;
- разрешенные домены.

Что должно получиться:

- сайт появился в списке;
- для него есть `siteCode`;
- для него настроен безопасный режим приема данных.

### Шаг 3. Включите безопасный сбор данных

Для сайта нужен signed same-site proxy path.

Логика подключения такая:

1. браузер отправляет данные не сразу в Метрику;
2. сайт принимает их на своих маршрутах;
3. same-site proxy подписывает запрос;
4. Метрика принимает только подписанные `collect` и `consent`.

Базовые маршруты на стороне сайта:

- `/api/analytics/collect`
- `/api/analytics/consent`

Upstream Метрики:

- `https://analytics.example.com/api/collect`
- `https://analytics.example.com/api/consent`

Для proxy нужны:

- `siteCode`
- signed ingest secret сайта
- collect URL панели
- consent URL панели

### Шаг 4. Подключите сам сайт

На стороне сайта настройте:

- локальный proxy для `collect`
- локальный proxy для `consent`
- фронтенд-трекер на эти same-site маршруты

Если ваш сайт уже работает на Next.js, самый прямой путь:

- локальный route handler на сайте;
- POST в `/api/analytics/collect` и `/api/analytics/consent`;
- передача `siteCode` в payload;
- проксирование в панель Метрики.

Если сайт на другом стеке, логика та же:

- same-site proxy остается на вашем домене;
- наружу не публикуется прямой unsigned browser-to-ingest путь.

### Шаг 5. Проверьте, что данные реально пошли

Проверьте по порядку:

1. страница сайта открывается;
2. consent banner отрабатывает;
3. после согласия есть запрос в `/api/analytics/consent`;
4. после просмотра страницы есть запрос в `/api/analytics/collect`;
5. в панели начинают появляться события и сводка.

## Что проверить после подключения

Чек-лист:

- панель открывается по HTTPS;
- владелец активирован и может войти;
- сайт добавлен в Метрику;
- `siteCode` задан корректно;
- collect и consent маршруты отвечают без ошибок;
- события поступают;
- согласия фиксируются;
- цели можно создать;
- обзор и отчетность заполняются.

## Частые ошибки и что делать

### Порт занят

Симптом:

- установщик пишет, что `80` или `443` уже заняты.

Что делать:

- проверьте, что уже слушает эти порты;
- остановите конфликтующую службу;
- повторите установку.

### Docker не стартует

Симптом:

- контейнеры не поднимаются;
- installer останавливается на Docker.

Что делать:

- проверьте состояние Docker;
- посмотрите install log;
- если Docker установлен вручную и сломан, сначала почините его, потом повторите запуск.

### Панель не открывается

Симптом:

- домен не отвечает;
- браузер не открывает панель.

Что делать:

- проверьте DNS;
- проверьте, что домен указывает на сервер;
- проверьте, что `80/443` открыты;
- откройте install log и итоговый отчет.

### Домен не смотрит на сервер

Симптом:

- установка прошла, но адрес панели не открывается снаружи.

Что делать:

- проверьте A/AAAA запись домена;
- дождитесь обновления DNS;
- повторно проверьте доступность панели.

### Данные не идут

Симптом:

- панель открывается, но событий нет.

Что делать:

- проверьте, что сайт создан в панели;
- проверьте `siteCode`;
- проверьте signed proxy на стороне сайта;
- проверьте, что запросы идут в `/api/analytics/collect` и `/api/analytics/consent`;
- проверьте, что consent действительно получен.

### Не проходит активация владельца

Симптом:

- ссылка не открывается;
- ссылка истекла;
- пароль не задается.

Что делать:

- проверьте, что используете актуальную ссылку из `owner-activation.txt`;
- если ссылка утеряна или истекла, обратитесь в Intellions.

### Ошибки collect / signature

Симптом:

- `401`
- `invalid_signature`
- `signature_required`
- `siteCode is outside of configured signed proxy scope`

Что делать:

- проверьте, что siteCode на сайте совпадает с siteCode в панели;
- проверьте, что signed ingest secret задан верно;
- проверьте, что браузер не шлет события напрямую в upstream без same-site proxy.

## Что входит в поддержку и когда обращаться в Intellions

Обычно вы можете пройти сами:

- подготовку сервера;
- запуск установщика;
- первый вход владельца;
- добавление сайта;
- базовую проверку панели.

Лучше писать в Intellions, если:

- не проходит активация владельца;
- не поднимается панель после успешной установки;
- не идут подписанные collect/consent запросы;
- нужен режим публикации через путь `/analytics`;
- требуется повторная выдача доступа владельца;
- нужен production-grade handoff для боевой среды.

## Честный статус этой инструкции

### Уже подтверждено

Подтверждено по текущей продуктовой реализации:

- универсальный one-command installer;
- image-first путь;
- activation-first flow владельца;
- installation identity;
- entitlement layer;
- signed ingest / consent contour;
- продуктовая панель и базовый контур управления.

### Что еще требует полного внешнего прогона

Эта инструкция уже описывает реальный продуктовый путь, но для полной внешней приемки еще нужен живой end-to-end прогон на отдельном клиентском сервере:

- чистый внешний Linux-хост;
- реальный домен клиента;
- полный first install outside internal contour;
- полная интеграция same-site proxy на внешнем сайте клиента.

Для первого боевого запуска по-прежнему полезно:

- идти по handoff-пакету, который вы получили вместе с продуктом;
- при необходимости подключить Intellions на этапе первого запуска и проверки сбора данных.
